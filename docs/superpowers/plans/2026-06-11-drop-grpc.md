# Drop gRPC Support Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Remove all gRPC artifacts from the Java subproject — deps, protoc plugin, `_services.proto` generation, gRPC stub tests, and all doc references — leaving protobuf model generation intact.

**Architecture:** gRPC currently exists only as generated stubs (side-effect of `modelsToProto` + `protoc-gen-grpc-java`) and tests verifying those stubs. No handwritten gRPC server code exists. Removal is: strip the plugin from `build.gradle`, delete the service-proto generation loop and method in `ModelsToProtoConverter`, delete all gRPC test methods (including a build-breaking one in `ConformanceTest`), delete the `appget_services.proto` golden fixture, then scrub all doc and skill references.

**Tech Stack:** Gradle (`java/build.gradle`), Java 25, JUnit 5. Build verified via `make test` and `make all` in `java/`.
